# Changelog: v3

Detailed release history for the `3.x` line.

## [3.11.1] - 2026-04-25

### Added

- Structured `SEARCH_NOT_CONFIGURED` errors for search misconfiguration, with
  stable `code` and `meta` fields.
- Reserved-key warnings for nested filters that accidentally place control keys
  like `page`, `limit`, `sort`, or `select` under `filters`.
- Framework-adapter-friendly return typing for
  `buildCrudSchemasFromModel` / `buildCrudSchemasFromMongooseSchema`.

### Documentation

- Clarified nested subdocument filtering in `README.md`.
- Condensed `skills/mongokit/SKILL.md`.

### Housekeeping

- Added governance and conformance tests.
- Audited repo-core / arc contract alignment and confirmed no further drift.

## [3.11.0] - 2026-04-22

### Added

- Promoted `updateMany` and `deleteMany` from plugin-only helpers to built-in
  `Repository` methods.
- Added portable `UpdateInput` IR handling to `findOneAndUpdate` and
  `updateMany`.

### Changed

- Renamed mongokit's single-document patch type to `UpdatePatch<TDoc>`, keeping
  the old name as a deprecated alias.
- Bumped the peer dependency floor to `@classytic/repo-core >= 0.2.0`.

### Housekeeping

- Added conformance coverage around the newly required standard repository
  methods.

## [3.10.3] - 2026-04-21

### Added

- `multiTenantPlugin` now respects tenant fields already present on input
  payloads instead of overwriting them.
- Expanded tests around data injection, strict mode, and `skipWhen` /
  `resolveContext` composition.

## [3.10.2] - 2026-04-20

### Fixed

- Full `StandardRepo<TDoc>` conformance across arc boundaries without casts.
- Widened `SelectSpec` mutability and session typing to match the repo-core
  contract.
- Flattened `LookupPopulateOptions` so it no longer inherited a conflicting
  index signature.

### Added

- A compile-time conformance gate to prevent future drift.

## [3.10.1] - 2026-04-20

### Fixed

- `Repository.lookupPopulate` now matches the `StandardRepo<TDoc>` signature,
  including generic result typing and wider filter support.

## [3.10.0] - 2026-04-20

### Breaking

- `getById`, `update`, and `delete` now return miss values instead of throwing
  by default. `throwOnNotFound: true` preserves the old behavior.
- `lookupPopulate()` now returns the standard pagination envelope and uses
  `docs` instead of `data`.
- `aggregate()` is now the portable aggregate IR. Raw Mongo pipelines moved to
  `aggregatePipeline()`, and paginated pipeline aggregation moved to
  `aggregatePipelinePaginate()`.
- `withTransaction()` now passes a transaction-bound repository (`txRepo`)
  instead of a raw session.
- `CacheAdapter.delete(key)` replaces `CacheAdapter.del(key)`.

### Added

- Portable lookup IR and portable filter IR support across repository methods.
- `Repository.bulkWrite(operations)` using the cross-kit contract.
- Cross-kit conformance test suites.

### Fixed

- `isDuplicateKeyError()` now recognizes wrapped `HttpError 409` shapes.

### Internal

- Migrated shared repository plumbing to `@classytic/repo-core`.

## [3.9.0] - 2026-04-17

### Added

- `Repository.isDuplicateKeyError(err)` as the authoritative duplicate-key
  classifier for MongoDB-backed repos.
- Top-level `isDuplicateKeyError` export aligned with the repository helper.

## [3.8.0] - 2026-04-17

### Added

- `Repository.findOneAndUpdate(filter, update, options)` as a first-class
  atomic update primitive.
- Plugin support for `findOneAndUpdate`, including multi-tenant, soft-delete,
  and timestamp handling.
- Central operation registry for plugin classification.

### Changed

- `findAll` hook context now uses `context.query` instead of `context.filters`.

## [3.7.0] - 2026-04-17

### Added

- Pagination hardening, including cursor versioning and stricter keyset sort
  controls.
- `createTenantContext()` to simplify tenant propagation with
  `AsyncLocalStorage`.
- Plugin-order validation at repository construction time.
- Cache TTL jitter controls.
- QueryParser depth and regex-complexity guards.
- Vector plugin URL allowlists and private-IP blocking controls.

### Changed

- Duplicate-key parsing is PII-safe by default.

### Housekeeping

- Added tiered Vitest projects and substantial new test coverage.

## [3.6.4] - 2026-04-15

### Fixed

- OpenAPI vendor extension output (`x-ref`) is now opt-in via
  `openApiExtensions: true`, restoring Ajv strict-mode compatibility by
  default.

### Added

- `openApiExtensions?: boolean` in `SchemaBuilderOptions`.

## [3.6.3] - 2026-04-15

### Fixed

- Query and schema generation issues around list-query types, coercion, and
  search delegation.
- Security issue where dangerous `$or` branches could degrade into match-all
  placeholders.
- Search resolver plugins can now truly clear `ctx.search`.

### Added

- QueryParser schema-aware value coercion.
- Repository-level `searchMode` / `searchFields` options with regex and auto
  modes.
- Documented search-resolver plugin contract for external search backends.
- Query primitive subpath exports and more composition coverage.

## [3.6.2] - 2026-04-15

### Added

- Optional `session` support for `getNextSequence()`, which lets transactional
  ID generators avoid sequence gaps on rollback.

## [3.6.1] - 2026-04-14

### Added

- `multiTenantPlugin.fieldType` to support ObjectId-backed tenant fields.

## [3.6.0] - 2026-04-12

### Added

- Unified `delete(id, { mode: 'hard' | 'soft' })` and matching `deleteMany`
  mode support.
- `before:restore` hook.
- Module-level `withTransaction(connection, callback, options)`.
- Smarter keyset index warnings based on declared schema indexes.
- Repo-routed cascade relations so target repository hooks and policies run.
- An outbox recipe built from existing hooks and transaction primitives.

### Changed

- Rewrote the README to a shorter, more practical format.
- Switched tests to a shared replica-set setup for real transaction coverage and
  faster runs.

### Fixed

- Cascade delete correctness for tenant scoping, custom deleted-field names, and
  target hook execution.

## [3.5.5] - 2026-04-07

### Added

- Query primitives as dedicated subpath exports.
- `PaginationEngine.paginate({ countFilters })`.
- Better architectural guidance in `SKILL.md`.
- QueryParser schema-aware coercion and repository search modes.
- A documented plugin contract for external search backends.

### Fixed

- Dangerous `$or` sanitization edge cases.
- Numeric equality coercion inconsistencies.
- Search resolver plugin clearing behavior.

### Internal

- Replaced internal warning calls with the shared logger helper and further
  refactored QueryParser internals.

## [3.5.4] - 2026-04-07

### Added

- `softRequired` schema support for fields that are database-required but
  request-body optional.

## [3.5.2] - 2026-04-04

### Added

- `idField` support for alternate identifiers.
- `getOne(filter, opts)`.
- `findAll(filters, opts)` with sort support.
- `maxLimit: 0` as unlimited.

### Changed

- `createMany()` now defaults to `ordered: false`.
- Invalid ObjectId lookups now return miss semantics instead of surfacing a
  cast error.

### Fixed

- Plugin support for custom `idField`.
- Pagination defaults now respect `0` values correctly.

## [3.4.1] - 2026-03-31

### Added

- Compound keyset sorts.
- Collation support.
- Keyset pagination for `lookupPopulate`.
- `lookupPopulate({ countStrategy: 'none' })`.

### Fixed

- Lookup count inflation.
- Lookup field projection bugs.
- Uncorrelated lookup joins in several pipeline paths.
- Keyset cursor parsing and sort validation edge cases.
- Unbounded lookup counts and incorrect `$expr` blocking.

### Changed

- Raised the Node.js requirement to `>=22`.
- Tightened aggregate pagination typing.
- Improved `RepositoryInstance` typing and exported `CollationOptions`.

## [3.4.0] - 2026-03-31

### Added

- Soft-delete-aware batch operations.
- `parseDuplicateKeyError()`.
- Parallel pagination via concurrent find/count.
- Automatic lookup routing in `getAll`.
- Lookup field-selection shorthand.

### Fixed

- `updateMany` now respects hook-modified `context.data`.

## [3.3.0] - 2026-03-16

### Fixed

- `deleteByQuery()` runtime crash.
- Pagination `hasNext` false positives when using `countStrategy: 'none'`.
- Boolean cursor serialization.
- `Repository.delete()` return typing and soft-delete response shape.
- Cascade hook registration race conditions.
- Several safety, cache, transaction, and custom-ID edge cases.

### Changed

- `CacheStats` now includes an `errors` counter.
- `DeleteResult` is the canonical return type for `Repository.delete()`.

## [3.2.1] - 2025-02-16

### Added

- Comprehensive `returnDocument: 'after'` integration coverage.

### Fixed

- Removed an unnecessary soft-delete type cast.

## [3.2.0] - 2025-02-16

### Added

- Vector search plugin and aggregation helpers.
- Multi-tenant plugin.
- Observability plugin.
- Centralized logger configuration.
- Hook removal APIs and expanded testing/security docs.

### Fixed

- Transaction retry behavior.
- Hook contract consistency for `getAll`.
- Cache key accuracy.
- Several pagination, vector, and error-handling edge cases.

### Security

- Tenant isolation for update/delete.
- Lookup pipeline sanitization and query parser hardening.

## [3.1.1] - 2024-12-XX

### Added

- Lookup aggregation support.
- Fallback transaction support.
- Improved TypeScript types.

### Fixed

- Minor bug fixes and cleanup.

## [3.0.1] - 2024-XX-XX

### Added

- Async event support.
- Cascade delete plugin.
- Enhanced soft-delete support with TTL.
- Comprehensive caching support.

### Changed

- Migrated the codebase to TypeScript.
